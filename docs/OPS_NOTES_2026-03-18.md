## Pi Customer / Job Sync

The Pi now has a working recurring sync from the Supabase mirror into the app's local DB cache.

### Source Tables

- customer table: `erp_mirror_cust`
- ship-to/job table: `erp_mirror_cust_shipto`

### Local App Behavior

- Synced records are stored in the local app DB table `customer_job_contexts`.
- Uploads use the local cache during matching, so live uploads do not depend on the cloud DB being online.

### Sync Script on Pi

- script path: `/home/amcgrean/services/list-ingestor/sync_customer_job_context.sh`
- log path: `/home/amcgrean/services/list-ingestor/logs/customer_job_sync.log`
